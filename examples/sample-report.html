Teknoloji Sürüm Özeti - 13 March 2026

MLflow

Sürümler: v3.10.1 (2026-03-05), v3.10.0 (2026-02-20)

Özet: v3.10 serisi multi‑workspace/organizasyon desteği, çok turlu değerlendirme ve konuşma simülasyonu, trace maliyet takibi ve yeniden tasarlanmış navigasyon/deneme (mlflow demo) gibi GenAI odaklı ve klasik ML iş akışlarını kolaylaştıran özellikler getirdi. v3.10.1 dikkat çekici hata düzeltmeleri ve workspace ile ilgili erişim/indirme sorunlarını giderdi.

  • Yeni özellikler (özellikle v3.10.0): Organizasyon/multi‑workspace, multi‑turn değerlendirme & oturum seviyesi scorer’lar, LLM span maliyet takibi, nav bar yeniden tasarımı (GenAI / Classical ML ayrımı), tek tıkla demo deneyimi, gateway kullanım takibi ve in‑UI trace değerlendirme.
  • Uyumluluk / Breaking notes: Resmi bir breaking change ilanı yok; ancak DB migration öncesi logged_model.experiment_id için CASCADE değişikliği ve workspace etkinse davranış değişiklikleri nedeniyle yedek/migration kontrolü önerilir.
  • Önemli düzeltmeler (v3.10.1): Workspace ile ilgili 403/indirme hataları, tracing/assessments.trace_id NOT NULL, MCP fn_wrapper ve span/çıktı işleme sorunları; UI modal/zaman seçici/tooltip/chart düzeltmeleri; Model Registry workspace arası copy hatası giderildi; SSO/OIDC dokümantasyon eklemeleri.

Detay & indirme: v3.10.1 release notları

Qdrant

Son sürüm: v1.17.0 (2026-02-20) — önceki: v1.16.3, v1.16.2

Özet: v1.17.0 ile Relevance Feedback, optimizasyon ilerleme raporları, cluster telemetri API'si, sınırsız güncelleme kuyruğu, Audit Access Logging ve Weighted RRF gibi özellikler eklendi; altyapı/performans ve snapshot/replication güvenilirliği üzerinde iyileştirmeler sürdürüldü. Önceki 1.16.x sürümleri kritik WAL/consensus düzeltmeleri içeriyordu.

  • Yeni/öne çıkan: Relevance Feedback, optimizasyon ilerleme raporu API, cluster telemetri toplama API'si, sınırsız update queue, Audit Access Logging, weighted RRF, upsert için update_mode, ikincil API anahtarı rotasyonu ve /metrics için ayrı port.
  • İyileştirmeler: Snapshot kurtarma optimizasyonları, syncfs flush iyileştirmeleri, okuma/güncelleme timeout geliştirmeleri.
  • Düzeltmeler: 1.16.2/1.16.3 zinciri kritik WAL/consensus veri bozulması risklerini giderdi; snapshot/replica transfer hataları, disk IO ve shard transfer/abort mantık hataları düzeltildi.
  • Breaking changes: Bu sürümlerde geriye dönük uyumluluğu bozan değişiklik bildirilmemiştir.

Detay & indirme: v1.17.0 release notları

OpenShift AI

Detay & indirme: v3.4.0-ea.2 release notları

Red Hat AI (InstructLab)

Son sürüm: v0.26.1 (2025-05-05) — önceki: v0.26.0, v0.26.0a1

Özet: Model family desteği, YAML tabanlı model id konfigürasyonu, Pydantic → YAML seri hale getirme iyileştirmeleri ve ek e2e/GPU testleri getirildi. Önemli olarak Python 3.10 desteği kaldırıldı ve bağımlılık kısıtlarında değişiklikler var; bu, çalışma zamanları ve paketleme davranışı üzerinde etkiler doğurabilir.

  • Yeni özellikler: model family alanı, YAML’den model id varsayılanlarını alma, Pydantic listelerinin YAML dökümü geliştirmeleri, yeni e2e/GPU test varyantları.
  • Breaking changes: Python 3.10 desteği kaldırıldı (en az Python 3.11/3.12 bekleniyor). Bağımlılık sınırları güncellendi (ör. torch <2.7.0, vllm >=0.8.0,<0.9.0) ve bazı autoconfig/torchscript davranışları değişti.
  • Düzeltmeler: v0.26.1'de CUDA kontrolüyle ilgili çökme düzeltildi; v0.26.0'da generation davranışı ve CLI/GPU rehberi iyileştirildi.

Detay & indirme: v0.26.1 release notları

Ray

Son sürüm: ray-2.54.0 (2026-02-18) — önceki: 2.53.0, 2.51.2

Özet: Ray Data tarafında checkpointing, genişletilmiş Compute Expression seti ve read_sql/sql_params desteği; yeni veri kaynakları (Kafka, Iceberg evrimi) ve V2 veri odaklı cluster autoscaler ile ölçekleme güncellemeleri öne çıkıyor. API kaldırmaları ve pydantic desteğiyle ilgili ileriye dönük uyarılar var.

  • Yeni özellikler: Ray Data checkpointing, çok sayıda Compute Expression, read_sql ve sql_params, Kafka datasource, Iceberg evrimi desteği; V2 autoscaler 2.54 ile varsayılan hale getirildi.
  • Breaking / dikkat: Pydantic V1 desteği gelecekte bırakılacak; 2.54 ile yeni autoscaler varsayılan olması davranış değiştirebilir; bazı deprecated API'ler kaldırıldı (ör. TENSOR_COLUMN_NAME, meta_provider, read_parquet_bulk vb.).
  • Düzeltmeler: CVE-2025-62593 dashboard header reddi düzeltmesi (2.51.2), sayısal kararlılık ve Iceberg yazma/okuma güvenilirliği iyileştirmeleri.

Detay & indirme: ray-2.54.0 release notları

Docker (Moby)

Son sürüm: docker-v29.3.0 (2026-03-05) — client/v0.3.0, api/v1.54.0

  • Yeni özellikler: --mount bind-create-src seçeneği, plugin hook'larının hata durumlarında da tetiklenmesi / error-hooks, API için identity parametreleri, BuildKit v0.28.0 güncellemesi.
  • Dikkat: Plugin davranışındaki genişleme ve AMD GPU için CDI tabanlı enjeksiyon deploy/CI süreçlerinde kontrol gerektirebilir.
  • Düzeltmeler: DNS konfigürasyonu reload sorunları, "rw layer snapshot not found", Windows docker top panic’i, sd_notify/systemd reload iyileştirmeleri ve gereksiz mount uyarıları azaltıldı.

Detay & indirme: docker-v29.3.0 release notları
